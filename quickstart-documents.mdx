This guide walks you through connecting your AI assistant to Moxn Documents so it can read and write documentation safely.

## What You'll Set Up

After this guide, your AI assistant will be able to:
- Read your team's documents for context
- Create and edit documents on branches
- Commit changes for you to review and merge

All writes happen on branches—your main content stays protected until you approve changes.

## Prerequisites

- A Moxn account (get one at [moxn.dev](https://moxn.dev))
- An AI assistant that supports MCP:
  - **Desktop apps**: Claude Desktop, ChatGPT Desktop, Cursor
  - **CLI tools**: Claude Code, Gemini CLI
  - **Cloud environments**: Claude Cowork

<Note>
**No coding required.** This setup involves copying a configuration snippet—that's it.
</Note>

## Step 1: Get Your Configuration

1. Sign in to [moxn.dev](https://moxn.dev)
2. From the Dashboard, find the **Knowledge Base** card
3. Click **MCP Setup**
4. Copy the configuration snippet

<Frame>
  <img src="/images/mcp-setup-dialog.png" alt="MCP Setup dialog showing configuration" />
</Frame>

Your configuration will look like this (with your workspace slug):

```json
{
  "mcpServers": {
    "knowledge-base": {
      "type": "stdio",
      "command": "npx",
      "args": [
        "@moxn/mcp-kb",
        "--workspace=your-workspace-slug"
      ]
    }
  }
}
```

## Understanding Connection Types

Not sure which setup to use? Here's the difference:

| Type | How It Works | Apps That Use It |
|------|--------------|------------------|
| **Local (stdio)** | Runs on your machine via `npx` | Claude Desktop, Claude Code, Cursor, Gemini CLI |
| **Remote (http)** | Connects to Moxn's servers | ChatGPT Desktop, Claude Cowork |

**Local is better when available.** It supports all features including file uploads, and keeps data flowing through your machine.

**Remote is required for cloud environments.** Apps like ChatGPT Desktop and Claude Cowork run in VMs that can't execute local commands. The `upload_url` tool doesn't work reliably in remote mode.

### Using Multiple AI Assistants

If you use several AI tools (e.g., Claude Code on your laptop and Claude Cowork in the browser), you can configure both connections. Use different server names so each tool knows which to use:

```json
{
  "mcpServers": {
    "knowledge-base": {
      "type": "stdio",
      "command": "npx",
      "args": [
        "@moxn/mcp-kb",
        "--workspace=your-workspace-slug"
      ]
    },
    "knowledge-base-remote": {
      "type": "http",
      "url": "https://your-workspace-slug.moxn.dev/api/mcp/http"
    }
  }
}
```

## Step 3: Authenticate

The first time you use Moxn Documents with your AI:

1. Ask your AI something like "List my Moxn documents"
2. A browser window will open for authentication
3. Sign in with your Moxn account
4. Return to your AI assistant

Your session persists across restarts—you only need to authenticate once.

## Try It Out

Once connected, try these prompts with your AI:

**Read documents:**
> "What documents do I have in Moxn?"

> "Read the API documentation from my knowledge base"

**Create content (safely on a branch):**
> "Create a new document called 'Meeting Notes' with today's discussion points"

> "Update the onboarding guide to include the new security requirements"

Your AI will create a branch for any changes. You can review and merge in the Moxn web app.

## How Safe Writes Work

When your AI creates or edits content:

1. **AI creates a branch** - Changes are isolated from your main content
2. **AI commits changes** - Work is saved on the branch
3. **You review in the web app** - See exactly what changed
4. **You merge when ready** - Approve changes to make them live

This means you can give your AI full write access without worrying about accidental overwrites or mistakes going live.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Claude Desktop doesn't show the hammer icon">
    - Make sure you saved the config file and restarted Claude Desktop completely (not just closed the window)
    - Check that the JSON is valid (no trailing commas, matching braces)
    - On Mac, you can find the config file at `~/Library/Application Support/Claude/claude_desktop_config.json`
    - On Windows, it's at `%APPDATA%\Claude\claude_desktop_config.json`
  </Accordion>

  <Accordion title="Authentication window doesn't open">
    - Make sure your default browser isn't blocking popups from Moxn
    - Try running the MCP server manually to see errors: `npx @moxn/mcp-kb --workspace=your-workspace-slug`
  </Accordion>

  <Accordion title="'npx' is not recognized">
    You need Node.js installed. Download it from [nodejs.org](https://nodejs.org) (choose the LTS version).

    After installing, restart your terminal/computer and try again.
  </Accordion>
</AccordionGroup>
