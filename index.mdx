## Built for Agents

Agents connect to Moxn via MCP and get structured access to your knowledge base through a standard tool interface: `find`, `read`, `search`, `documents`, and `sections`.

Unlike bolting an API onto a wiki, Moxn is built from the ground up for agent access. Documents are organized by paths, sections are individually addressable, and search returns clean text — not HTML fragments or PDF chunks.

## A Real Editor for Your Team

Moxn isn't just an API for agents — it's a full-featured editor for your team. Rich text, code blocks with syntax highlighting, mermaid diagrams, inline comments, and drag-and-drop section reordering.

<Frame>
  <img src="/images/kb-editor.png" alt="Moxn document editor with sections, code blocks, and mermaid diagrams" />
</Frame>

Your team writes and edits documentation in the web app. Agents read and write the same content over MCP. Everyone works from a single source of truth.

## Version Control Built In

Every document has branches, commits, and merge requests — the same workflow developers already know from Git.

- **Branch** to experiment or isolate agent writes
- **Commit** to create immutable snapshots
- **Merge** when you're satisfied with the changes
- **Roll back** to any previous version
