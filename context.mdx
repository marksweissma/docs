# Why Moxn?

AI agents are useful when they have context. The problem is getting your team's knowledge into a format agents can actually use.

## The Problem

Most teams keep docs in wikis, Google Drive, Notion, or markdown files. To give an agent access you end up:

- Copy-pasting content into prompts
- Building RAG pipelines over exports
- Writing custom integrations per tool

And when agents need to *write* docs, it's worse — either they can't, or they edit production content directly with no review step.

## How Moxn Works

Moxn is a context OS with a built-in MCP server. Your agents connect and get structured access to your docs through a standard tool interface.

**For reading:** Agents use `find`, `read`, and `search` tools to query your content. No RAG pipeline, no embeddings, no exports — agents read the same docs your team writes in.

**For writing:** When you ask an agent to create or update content, it works on a branch. You review the diff and merge, just like a pull request. Main is always protected.

**For permissions:** You control what agents can see at the section level. API keys, credentials, internal metrics — mark sections as hidden from LLMs and they're filtered from MCP responses.
