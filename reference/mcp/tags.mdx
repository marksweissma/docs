Manage tag definitions. Tags are hierarchical labels used to categorize documents (e.g., `/technology/ai/google`). This tool manages the **catalog** ‚Äî the definitions themselves. To assign or remove tags on documents, use the [`documents` tool](/reference/mcp/documents).

<Note>
New to tags? See [Tags](/concepts/tags) for the mental model ‚Äî what tags are, how hierarchy works, and how they differ from folders.
</Note>

## Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `action` | enum | Yes | `list`, `create`, `delete`, or `update` |
| `tagId` | UUID | Conditional | Required for `delete` and `update` |
| `tagPath` | string | Conditional | Required for `create` ‚Äî e.g., `"/technology/ai/google"` |
| `tagDescription` | string \| null | No | Description for `create` or `update` |
| `tagColor` | string \| null | No | Hex color for `create` or `update` (e.g., `"#3B82F6"`) |
| `tagDisplayName` | string \| null | No | Human-readable name ‚Äî shown in UI instead of path segment. For `create` or `update` |
| `filesystem` | string | No | Filesystem slug ‚Äî optional for `create` when you have multiple filesystems |

## Actions

| Action | tagId | tagPath | Notes |
|--------|-------|---------|-------|
| `list` | ‚Äî | ‚Äî | Returns all tags in the workspace |
| `create` | ‚Äî | REQUIRED | Auto-creates ancestor tags that don't exist yet |
| `delete` | REQUIRED | ‚Äî | Cascades ‚Äî deletes all descendants and removes all document assignments |
| `update` | REQUIRED | ‚Äî | Update color, description, or displayName |

## Tag Hierarchy

Tags are organized as paths using `/` separators. When you create `/technology/ai/google`, Moxn auto-creates `/technology` and `/technology/ai` if they don't exist.

This hierarchy is used for:
- Organizing the tag catalog visually
- Backing database column options (select, multi-select, status, checkbox columns all reference tag IDs)
- Filtering in `find` via `tagIds`

## Response (list)

```json
[
  {
    "id": "uuid",
    "path": "/technology/ai",
    "displayName": "AI",
    "description": "Artificial intelligence topics",
    "color": "#3B82F6"
  }
]
```

## Examples

```
"Create a 'needs-review' tag"
‚Üí tags({action: "create", tagPath: "/status/needs-review", tagColor: "#F59E0B", tagDisplayName: "Needs Review"})

"List all tags"
‚Üí tags({action: "list"})

"Rename the display name of a tag"
‚Üí tags({action: "update", tagId: "...", tagDisplayName: "Under Review"})

"Delete the /drafts tag and all its children"
‚Üí tags({action: "delete", tagId: "..."})
```
